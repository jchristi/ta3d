Les commandes de la Console

Appuyez sur ")" sous Linux ou sur "`" ou "²" sous windows pour afficher/cacher la console pendant le jeu.
(n est un entier, f un nombre en virgule flottante)

Commandes de contrôle de l'interface

fps_on/fps_off : afficher/cacher le nombre d'images par seconde
freecam : active/désactive le mode caméra libre
enable right_click_interface: active l'interface clic droit
enable left_click_interface: active l'interface clic gauche

Commandes de contrôle du Gameplay


zshoot : capture du tampon de profondeur
video shoot : active/désactive l'enregistrement video
reload shaders : recharge les vertex et fragment programs utilisés pour le rendu de l'eau et des détails
show mission info : affiche les informations de mission et les drapeaux de l'unité sélectionnée
view debug : affiche sur la minicarte les zones dessinées par le moteur de rendu
ai debug : affiche des informations sur l'IA pour déboguer
internal name : affiche/cache le nom interne des unités
internal idx : affiche/cache le numéro de l'unité sélectionnée
exit : quitte le jeu
wireframe : active/désactive le mode fil de fer
priority n : 0, 1 ou 2, défini la priorité du programme (0:basse, 1:moyenne, 2:haute)
water quality n : de 0 à 5 défini le niveau de détails du rendu de l'eau (0:faible, 1:normal, 2:haute, 3:très haute, 4:extrème, comme 3 mais avec tout les reflets, 5: ultime)
shadow quality n : qualité des ombres (n=1: normal, plus n est grand, plus les ombres sont détaillées au détriment des performances)
shadow ray f : diffusion des ombres (nécessite une qualité d'ombrage suffisante)
shadow : active/désactive les ombres
details : (dés)active la texture de détails
particle : affiche/cache les particules
waves : affiche/cache les vagues sur les côtes
show script : affiche/cache les scripts liés aux unités sélectionnées
show model : affiche/cache les informations sur le modèle 3d de l'unité sélectionnée
rotate light : active/désactive la rotation de la lumière
shake : fait trembler l'écran comme lors d'une grosse détonation
fps_limit n : impose un fps maximum (n<=0 : aucune limite)
spawn nom_de_l_unité id_joueur nb_unités : crée les unité(s) spécifiées pour le joueur indiqué. Syntaxe:
nom_de_l_unité - spécifie le type d'unité par son nom, pratique pour tester ("any" = aléatoire)
id_joueur - spécifie le joueur qui controlera ces unités
nb_unités - nombre d'unités à créer
timefactor f : change la vitesse du jeu (1 = normal)
addhp n : ajoute n points de vie aux unités sélectionnées (n peut être négatif)
activate/deactivate : active ou désactive les unités sélectionnées
reset_script : arrête tous les scripts en cours d'éxécution pour les unités sélectionnées
unitinfo : affiche des informations sur les unités sélectionnées dans une console
kill : tue les unités sélectionnées
script n : lance le script numéro n sur chaque unité sélectionnée
pause : pause/reprendre la partie
give metal/energy/both player_id amount : donne les ressources amount au joueur player_id
metal : active/désactive le code triche du métal
energy : active/désactive le code triche de l'énergie
